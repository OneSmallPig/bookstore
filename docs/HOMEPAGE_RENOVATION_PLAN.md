# 首页推荐体系整改方案

## 目标

本次整改目标是将首页从“4本推荐 + 静态分类 + 热门书籍”改造成“基于场景的首页推荐聚合页”。

未登录场景：

- 首页推荐固定返回 8 本书
- 传统文学 4 本
- 小说 4 本
- 首页分类固定展示 6 个分类
- 分类由 AI 综合分析生成
- 移除热门书籍板块

登录场景：

- 首页推荐固定返回 8 本书
- 依据用户书架和阅读行为生成推荐
- 首页分类固定展示 6 个分类
- 分类依据用户阅读情况分析生成
- 移除热门书籍板块

## 现状问题

### 前端

- 首页推荐当前只请求 4 本
- 首页“热门分类”是静态 HTML
- 首页“热门书籍”是单独动态接口
- 首页推荐和分类并不共享同一套用户画像
- 首页有缓存，但缓存对象只覆盖推荐/热门书籍，不覆盖完整首页数据

### 后端

- 推荐接口只返回单一书单，没有首页聚合结构
- 当前个性化逻辑依赖 `user.readHistory`，但实际用户对象中没有这个字段
- 推荐失败时会回退模拟数据，但前端无法知道返回来源
- 登录态和未登录态没有统一的首页响应结构

## 设计方案

### 一、首页聚合接口

新增首页专用接口：

- `GET /api/ai/homepage`

接口特性：

- 公共可访问
- 如果请求头里带有效 token，则按登录态处理
- 如果没有 token，则按未登录态处理
- 统一返回首页推荐和分类

建议响应结构：

```json
{
  "success": true,
  "data": {
    "scene": "guest",
    "source": "ai",
    "recommendationGroups": [
      {
        "key": "classic_literature",
        "title": "传统文学",
        "description": "适合建立稳定阅读底盘的文学经典",
        "books": []
      },
      {
        "key": "novel",
        "title": "小说精选",
        "description": "适合沉浸式阅读的叙事作品",
        "books": []
      }
    ],
    "categories": [
      {
        "name": "文学",
        "reason": "适合当前首页推荐氛围"
      }
    ]
  }
}
```

### 二、未登录态策略

推荐：

- 固定两组
- 传统文学 4 本
- 小说 4 本

分类：

- 固定返回 6 个分类位
- 由 AI 基于平台综合阅读取向生成

### 三、登录态策略

推荐：

- 从用户书架提取阅读画像
- 画像数据包括：
  - 最近阅读书籍
  - 阅读中书籍
  - 已完成书籍
  - 高频分类
  - 偏好作者
  - 阅读节奏信号
- 基于画像推荐 8 本书

分类：

- 固定返回 6 个分类位
- 根据阅读画像生成下一阶段更可能感兴趣的分类

### 四、回退策略

如果 AI 调用失败或解析失败：

- 未登录态回退到固定示例数据
- 登录态回退到“基于画像偏好排序”的本地示例数据
- 返回 `source: "fallback"`

### 五、前端改造

首页结构调整：

- 保留欢迎横幅
- 推荐区改为“分组推荐区”
- 分类区改为动态渲染
- 删除热门书籍板块

首页数据流调整：

- 首页只请求一次 `/api/ai/homepage`
- 用一份缓存保存完整首页数据
- 缓存与登录状态绑定

页面展示建议：

- 推荐区顶部展示来源标签，如“AI推荐”或“推荐回退”
- 推荐区按分组渲染
- 分类区展示 6 张分类卡片
- 每个分类卡片支持显示分类名和简短原因

## 实施步骤

1. 新增首页聚合接口与可选鉴权逻辑
2. 在 AI 服务层新增首页推荐聚合能力
3. 实现用户阅读画像提取
4. 实现未登录和登录两套首页提示词
5. 实现首页回退数据与来源标记
6. 改造首页前端容器与缓存逻辑
7. 删除热门书籍板块
8. 完成联调与验证

## 风险与注意点

- 当前开发环境存在代理配置，AI 实际请求可能被本地代理拦截
- 如果用户已登录但书架为空，需要定义回退行为
- 当前首页文件较大，建议本次只改首页相关入口，不顺带重构整份脚本
- 推荐结果仍依赖大模型格式稳定性，因此必须保留本地回退逻辑

## 本次实现边界

本次整改优先完成：

- 首页聚合接口
- 登录/未登录两套推荐逻辑
- 动态分类
- 移除热门书籍
- 前端首页新渲染与缓存

本次暂不处理：

- 更细粒度的推荐解释面板
- 首页推荐结果埋点统计
- 多轮用户兴趣学习
